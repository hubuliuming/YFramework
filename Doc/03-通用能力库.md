# 通用能力库

## 范围

本篇覆盖以下目录：

- `Assets/YFramework/Kit`
- `Assets/YFramework/Extension`
- `Assets/YFramework/Collections`
- `Assets/YFramework/Math`
- `Assets/YFramework/Msg`

这些代码共同构成 YFramework 的“工具层”。

## 1. 调度、延迟与计时

### `ActionKit`

文件：

- `Assets/YFramework/Kit/Action/ActionKit.cs`
- `Assets/YFramework/Kit/Action/ActionSpan.cs`

已明确实现的能力：

- `Delay(Action, float)`：延迟若干秒执行
- `DelayOneFrame(Action)`：下一帧执行
- `DelayOneFrame(Action, MonoBehaviour)`：在指定 Mono 上下一帧执行
- `SecondsFixedUpdate(int frequency, Action)`：创建一个临时 `ActionFixedUpdate` 节点，在固定时间窗口内按频率执行回调

关键依赖：

- `Delay` / `DelayOneFrame` 通过协程运行，默认挂在 `MonoGlobal.Instance` 上

注意：

- `ActionSpan` 本身当前基本为空
- 真正有逻辑的是 `ActionFixedUpdate`

### `TimerManager`

文件：

- `Assets/YFramework/Kit/Time/TimerManager.cs`
- `Assets/YFramework/Kit/Time/TimerKit.cs`

实际可用核心是：

- `TimerManager : MonoSingleton<TimerManager>`
- `TimerItem`

支持能力：

- 注册计时任务
- 暂停 / 恢复 / 停止 / 重启
- 周期更新回调
- 结束回调

使用特征：

- `TimerItem` 保存 `OnUpdate`、`onEnd`、`interval`、`endTime`
- `TimerManager.Update()` 逐个推进任务

注意：

- `TimerKit.cs` 当前基本是占位文件
- 真正入口应该理解为 `TimerManager + TimerItem`
- `TimerManager` 内部有一个共享 `_timer`，意味着多个不同 `interval` 的定时任务设计上要谨慎验证

## 2. 单例体系

### `MonoSingleton<T>`

文件：

- `Assets/YFramework/Kit/Singleton/MonoSingleton.cs`

特点：

- 适用于 `MonoBehaviour`
- 找不到实例时自动创建同名 `GameObject`
- `Awake()` 中 `DontDestroyOnLoad`

适合：

- 需要常驻场景的管理器
- 需要 `Update` / 协程 / 生命周期的工具类

### `NormalSingleton<T>`

文件：

- `Assets/YFramework/Kit/Singleton/NormalSingleton.cs`

特点：

- 约束 `where T : class, new()`
- 直接 `new T()` 构造实例
- 如果误用于 `MonoBehaviour`，会输出错误日志

适合：

- 纯逻辑对象
- 不依赖 Unity 生命周期的管理类

## 3. 网络封装

目录：

- `Assets/YFramework/Kit/Net`

### `TcpClient`

定位：

- 轻量 TCP 客户端封装

特征：

- 后台线程建立连接
- `BeginReceive` 异步接收
- 支持字符串或字节数组发送
- 支持 `onReceived` 回调

### `TcpServer`

定位：

- 轻量 TCP 服务端模板

特征：

- 后台线程启动监听
- `BeginAccept` 接收客户端
- 每个客户端连接后继续 `BeginReceive`
- 提供 `onReceved` 回调

附带说明：

- 同文件中的 `EncodeTool` 已被标记为 `[Obsolete(..., true)]`
- 可以认为该类无效，不应继续使用

### `UDPClient`

定位：

- 简单 UDP 收发封装

特征：

- 本地 socket 负责接收
- 远端 socket 负责发送
- 支持字符串或字节数组发送
- 提供 `OnReceived` 回调

统一注意：

- 这些网络类都比较底层，没有做线程调度、重连、主线程派发、协议拆包等高级封装
- 异常处理比较直接，部分地方会 `throw`
- 关闭时使用了 `Thread.Abort()` 风格，属于较旧实现

## 4. 点击事件与 EventSystem 辅助

目录：

- `Assets/YFramework/Kit/PointEvent`
- `Assets/YFramework/Kit/Utility/EventSystemUtility.cs`
- `Assets/YFramework/Kit/Utility/UIUtility.cs`

### `BaseButton` / `ButtonEvent` / `Button2DEvent`

定位：

- 给 2D / 3D 物体提供类似按钮点击能力

关系：

- `BaseButton` 实现 `IPointerClickHandler`
- `ButtonEvent` 在 `Start()` 中声明自己是 3D 对象
- `Button2DEvent` 在 `Start()` 中声明自己是 2D 对象
- `EventUtil.CheckRaycaster()` 会自动检查并补 `PhysicsRaycaster` 或 `Physics2DRaycaster`

这套逻辑的目的不是 UI Button 替代，而是给非 UI 物体补点击支持。

### `PEListener`

文件：

- `Assets/YFramework/UI/Runtime/EventSystem/PEListener.cs`

虽然位置在 UI 目录，但本质是一个通用指针事件分发器。
它对外暴露：

- `onClick`
- `onClickDown`
- `onClickUp`
- `onDrag`

适合在需要快速挂接指针事件时使用。

### `EventSystemUtility`

能力包括：

- `ExecuteClickAll(...)`：把当前点击继续分发给命中的其他对象
- `IsPointerOverUIElement()`：判断当前指针是否落在 UI 上

### `UIUtility`

能力：

- 基于 `GraphicRaycaster` 判断当前鼠标位置是否命中 UI

## 5. 文件、类型与基础工具

### 文件工具 `YFile`

文件：

- `Assets/YFramework/Kit/IO/YFile.cs`

当前实现包含：

- 文件重命名
- 按前缀规则分类文件到子目录

更偏编辑器/离线工具用途，不像运行时热路径能力。

### 类型解析 `TypeResolverForUnity`

文件：

- `Assets/YFramework/Kit/Type/TypeResolver.cs`

定位：

- 在 Unity 当前已加载程序集里解析类型

典型用途：

- AutoBind 根据字符串类型名寻找实际 `Type`

这是 AutoBind 能支持可选 TMPro 类型的关键基础设施之一。

### Utility 子目录

目录中包含一批常用静态工具，例如：

- `ArrayUtility`
- `EnumUtilty`
- `MathUtility`
- `TextFormat`

这类代码基本属于零散但常用的帮助类，强耦合不高。

## 6. 扩展方法库

目录：

- `Assets/YFramework/Extension`

这是项目中非常高频的一层，主要用来补齐 Unity 常见对象的便捷操作。

### 重点是 `TransformExtension`

文件：

- `Assets/YFramework/Extension/MonoBase/TransformExtension.cs`

它提供了大量便捷方法，例如：

- 改位置 / 旋转 / 缩放的单轴值
- `SetIdentity()` / `Reset()`
- `GetOrAddComponent<T>()`
- 获取激活子物体
- 方向判断 `HorizonDirection(...)`

风格上是明显的“减少样板代码”工具层。

### 其他扩展

还包括：

- `GameObjectExtension`
- `ComponentExtension`
- `AnimatorExtension`
- `EngineExtension`
- `CollectionsExtension`
- `VectorExtension`
- `StringExtension`
- `TxtExtension`
- `LineRendererExtension`

遇到“项目里为什么会有某个看起来像 Unity 原生 API 的便捷方法”时，优先从这里找。

## 7. 可序列化容器

目录：

- `Assets/YFramework/Collections`

### `SerializableDictionary<TKey, TValue>`

作用：

- 让字典以 `keys + values` 的形式参与 Unity 序列化

特点：

- 反序列化时会跳过空 key 和重复 key
- 如果键值数量不一致，会按较短长度恢复

### `SerializableKeyValue<T, V>`

作用：

- 用双列表持久化键值对列表

在项目中的重要用途：

- AutoBind 用它保存成员名映射、数组成员映射、类型映射

### `SerializableList`

定位：

- 针对 Unity 序列化场景的简单可序列化列表封装

## 8. 定点数数学

目录：

- `Assets/YFramework/Math`

核心类型：

- `Fixed64`
- `Fixed64Vector2`
- `Fixed64Vector3`

定位：

- 提供基于 `long` 的定点数与向量结构

特点：

- `Fixed64` 用 `Fractionalbits = 12`
- 重载了基础四则运算与比较操作
- `Vector2/Vector3` 是围绕 `Fixed64` 组合出的轻量数据结构

适合：

- 需要可控数值表达的逻辑层

当前观察：

- 这里只是数学结构本身，没有看到更完整的定点物理或大规模确定性系统

## 9. 当前不应过度解读的部分

### `Msg.cs`

文件：

- `Assets/YFramework/Msg/Msg.cs`

当前是空类。
不要把它当作已完成的消息系统入口。

- `Convert`

## AI 修改建议

如果你要在这层加功能，优先先选对位置：

- 要补 Unity 对象便捷方法：放 `Extension`
- 要补纯静态工具：放 `Kit/Utility`
- 要补运行时管理器：考虑 `Kit` + `MonoSingleton`
- 要补可序列化数据结构：放 `Collections`
- 要补数值结构：放 `Math`

如果不确定该放哪，一般先判断它是：

- 约定层能力
- 工具层能力
- UI 能力
- 编辑器能力

YFramework 当前的目录分层大致就是按这个逻辑组织的。
