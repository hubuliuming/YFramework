# UI 系统

## 范围

本篇覆盖：

- `Assets/YFramework/UI`
- 与 UI 强相关的扩展与编辑器支持

重点文件：

- `Assets/YFramework/UI/Runtime/UIBase.cs`
- `Assets/YFramework/UI/Runtime/Custom/Image/CircleImage.cs`
- `Assets/YFramework/UI/Runtime/Custom/Image/PolygonColliderImage.cs`
- `Assets/YFramework/UI/Runtime/Adaptive/SlideScrollHorizontal.cs`
- `Assets/YFramework/UI/Layout/RawImageAdaptivityLayout.cs`
- `Assets/YFramework/UI/Runtime/EventSystem/PEListener.cs`

## 整体定位

YFramework 的 UI 层不是大型窗口框架。
它更接近下面这个组合：

- 一个很轻的页面基类
- 一个简单的 UI 显示栈
- 一批自定义 UI 组件
- 一批布局和交互辅助脚本
- 一些方便在编辑器里创建 UI 控件的菜单项

## 1. `UIBase` 与 `UIManager`

文件：

- `Assets/YFramework/UI/Runtime/UIBase.cs`

### `UIBase`

`UIBase` 继承自 `YMonoBehaviour`，并实现 `IUIBase`。
它暴露两个用户层入口：

- `Show()`
- `Hide()`

同时要求子类实现：

- `OnShow()`
- `OnHide()`

### `UIManager`

`UIManager` 是一个非 Mono 的简单单例类。
它内部维护：

- 当前 UI：`CurUI`
- 历史 UI 栈：`_lastUIs`

当前逻辑更像“单窗口 + 返回栈”：

- `ShowUI(GameObject)`：打开一个 UI，如果已有当前 UI，就把旧 UI 压栈
- `HideUI()`：关闭当前 UI，并尝试回退到上一个
- `HideAllUI()`：持续关闭直到栈清空

### 重要结论

这不是完整的多层窗口管理器。
它没有：

- 面板路由
- 异步资源加载
- 层级系统
- 窗口配置表
- 模态遮罩管理

如果业务复杂度上升，通常要在此基础上继续封装。

## 2. UI 自定义图形组件

### `CircleImage`

文件：

- `Assets/YFramework/UI/Runtime/Custom/Image/CircleImage.cs`

定位：

- 继承 `UnityEngine.UI.Image`
- 通过重写 `OnPopulateMesh` 动态生成圆形网格
- 同时重写 `IsRaycastLocationValid`，实现圆形区域射线检测

功能特征：

- 用 `segments` 控制圆的分段数
- 通过顶点计算生成圆面
- 维护 `_vertexPos` 供点击命中判断使用

典型用途：

- 圆形头像
- 圆形进度视觉
- 非矩形点击区域

### `PolygonColliderImage`

文件：

- `Assets/YFramework/UI/Runtime/Custom/Image/PolygonColliderImage.cs`

定位：

- 继承 `Image`
- 依赖同物体上的 `PolygonCollider2D`
- 用碰撞体形状决定 UI 点击有效区域

适合：

- 不规则图片点击
- 比圆形更复杂的命中区域

## 3. 自适应与布局辅助

### `SlideScrollHorizontal`

文件：

- `Assets/YFramework/UI/Runtime/Adaptive/SlideScrollHorizontal.cs`

定位：

- 一个横向分页滚动辅助脚本

主要能力：

- 记录当前页 `currentIndex`
- 通过拖拽左右切页
- 支持上一页 / 下一页按钮
- 支持页码文本
- 支持 `JumpToPage(int)`

依赖：

- `ScrollRect`
- `GridLayoutGroup`

适用场景：

- 横向卡片页
- 简单轮播页
- 分页道具列表

### `RawImageAdaptivityLayout`

文件：

- `Assets/YFramework/UI/Layout/RawImageAdaptivityLayout.cs`

定位：

- 让 `RawImage` 在指定显示区域内按比例适配

关键特征：

- `[ExecuteAlways]`
- `Update()` 中持续执行适配
- 实际调用的是 `UIExtension.AdaptivitySize(...)`

用途：

- 编辑态 / 运行态都保持图片适配

### `UIExtension`

文件：

- `Assets/YFramework/UI/Extension/UIExtension.cs`

提供的高频能力包括：

- `ScrollRect.UpdateContentLength(...)`
- `RectTransform.AdaptivitySize(...)`
- `HorizontalLayoutGroup.SetCenterSize()`

这些方法主要服务于：

- Grid + ScrollRect 动态内容尺寸计算
- 图片按区域缩放
- 水平布局的内容居中

## 4. 指针事件辅助

### `PEListener`

文件：

- `Assets/YFramework/UI/Runtime/EventSystem/PEListener.cs`

它实现了：

- `IPointerClickHandler`
- `IPointerDownHandler`
- `IPointerUpHandler`
- `IDragHandler`

作用是把 Unity 事件接口转成可直接赋值的委托字段：

- `onClick`
- `onClickDown`
- `onClickUp`
- `onDrag`

适合：

- 临时 UI 交互接线
- 不想额外写一个专门组件类时的快速交互代理

## 5. UI 编辑器支持

UI 层不只有运行时脚本，还配有专门的编辑器增强。
详细见 `05-编辑器工具.md`，这里先记住两类入口：

- 创建 UI 控件的菜单项
- 自定义 Inspector

例如：

- `GameObject/YFreamwork/UI/CircleImage`
- `GameObject/YFreamwork/UI/PolygonColliderImage`

## 6. UI 层与 AutoBind 的关系

YFramework 的 UI 代码与 AutoBind 天然契合。
这是因为：

- `UIBase` 继承 `YMonoBehaviour`
- `AutoBindRules` 内置了大量 UI 前缀
- 测试样例里也以 UI 控件绑定为主

所以当你看到下面这些节点名时，通常意味着它们预期走 AutoBind：

- `BtnClose`
- `TxtTitle`
- `ImgAvatar`
- `TogMusic`
- `RectContent`

## 7. 当前边界与不足

### 1. 没有形成大一统 UI 框架

当前更像“轻基类 + 小工具集合”，不宜误判成完整窗口系统。

### 2. `UIManager` 是单栈思路

如果多个窗口并行、分层、模态化，会超出当前设计能力。

### 3. 部分脚本是强场景化组件

例如 `SlideScrollHorizontal` 假设了较具体的 `ScrollRect + GridLayoutGroup + 页码按钮` 结构。

### 4. 目录里存在非脚本元数据痕迹

`Assets/YFramework/UI/Runtime/Adaptive/.idea` 这类目录不属于运行时设计的一部分，阅读时可以忽略。

## AI 修改建议

如果你要扩展 UI，建议先判断是哪一类需求：

- 面板显示与切换：先看 `UIBase` / `UIManager`
- 非矩形点击：先看 `CircleImage` / `PolygonColliderImage`
- 内容区尺寸适配：先看 `UIExtension`
- 分页滚动：先看 `SlideScrollHorizontal`
- 快速指针事件：先看 `PEListener`

如果需要做更大型的 UI 框架化建设，建议把现有代码当作基础组件库，而不要把它误当成已完成的整套 UI 架构。
